Windshock Lens

Browser-side phishing protection for gray-zone pages in the AI era.

Windshock Lens combines local page extraction, deterministic security rules, OCR, domain ownership signals, and on-device Gemini Nano analysis to help evaluate suspicious pages without sending page content to external LLM APIs.

View on GitHub How it works
100% local page-content analysis Browsing context stays on the device.
Complements Safe Browsing Focuses on zero-hour and targeted gray-zone pages.
Rules and AI together Hard evidence is not reduced to a generic score.

Origin

Built from real defensive observations

Modern phishing pages abuse normal infrastructure, imitate trusted software, and adapt their login screens after rendering. Windshock Lens was shaped by field observations of AI client impersonation, fake software download pages, and enterprise sign-in phishing kits.

AI client impersonation

Pages that mimic popular AI tools or installers to capture trust before a download or login flow.

Fake software downloads

Lookalike download journeys that can push suspicious files or deceptive instructions.

Enterprise sign-in kits

Rendered login pages with reusable kit behavior, prefilled identity hints, and off-origin collection paths.

Normal infrastructure abuse

Phishing hosted on shared platforms, workers, pages, or static hosting that reputation systems may not classify yet.

Positioning

A browser-side triage layer

Windshock Lens is not a replacement for Chrome Safe Browsing, a cloud scanner, a generic antivirus, or a thin LLM wrapper. It is a local triage layer for suspicious rendered pages that need evidence-aware judgment before the user continues.

  • Zero-hour phishing and spear-phishing
  • Brand impersonation with credential forms
  • Fake download and clipboard-abuse flows
  • Rendered-page behavior visible only after load

Workflow

How Windshock Lens evaluates a page

The extension keeps page inspection inside the browser and combines direct evidence with contextual analysis only when needed.

  1. 1

    Inspect link, page, or download

    A clicked link, current page, or suspicious download starts a local scan.

  2. 2

    Extract rendered signals

    DOM, forms, scripts, OCR text, and domain ownership signals are collected without external LLM calls.

  3. 3

    Separate hard evidence from ambiguity

    Strong markers such as phishing-kit behavior or dangerous clipboard payloads should be handled deterministically.

  4. 4

    Use on-device AI for context

    Gemini Nano assists when the page is suspicious but still requires language, layout, or intent judgment.

  5. 5

    Warn, explain, or allow reset

    Users see a warning or detailed verdict and retain control over allow and reset decisions.

Detection examples

Evidence the extension can reason about

Public examples are intentionally sanitized. The project does not publish live phishing URLs, credentials, internal hostnames, or sensitive incident details.

Pattern Why it matters
Brand-like login page on a non-official domain May indicate credential harvesting or targeted impersonation.
Password form plus phishing-kit marker Combines credential collection with direct implementation evidence.
Hidden or prefilled corporate email hint Can indicate spear-phishing or account-specific lures.
Dynamic logo or background from a victim domain Common behavior in reusable phishing kits that adapt branding.
Fake download or clipboard shell payload Can lead to malware delivery or ClickFix-style command abuse.
Webhook endpoint combined with a credential form Suggests a possible exfiltration path while avoiding collection of secret values.

Design principles

Privacy-first, evidence-aware protection

No external LLM API for page content

The extension is designed so browsing context is analyzed on the device.

Rules are not replaced by AI

Deterministic evidence remains explicit and explainable.

AI handles ambiguous context

Language, layout, and intent judgment are useful when hard evidence is not enough.

Users keep control

Allowlist and page reset flows remain visible user decisions.

Status and roadmap

Experimental proof of concept

Windshock Lens is intended for security research, pilot testing, and defensive evaluation. It is not yet an enterprise-managed security product.

  • Add hard-evidence precheck before LLM analysis.
  • Expand phishing-kit markers for webhook and messaging-platform exfiltration patterns.
  • Design privacy-first endpoint-only network observation.
  • Review web-accessible resources and conservative safe-domain behavior.
