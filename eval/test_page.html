🛡 피싱 링크 검사기 — 수동 테스트

5가지 트리거와 사용법
  • 컨텍스트 메뉴 — 아래 링크를 우클릭 → 이 링크 피싱 검사 → 시스템 알림으로 결과
  • 툴바 액션 — 의심 URL로 이동 후 확장 툴바 아이콘 클릭 → 현재 페이지 검사 → 위험이면 탭이 warning 페이지로 전환
  • 다운로드 자동 — 다운로드 시작 시 호스팅 페이지 자동 검사 (이 페이지의 .exe 링크 클릭으로 테스트)
  • 클릭 가드 — 다운로드/Copy 버튼/위험 URI 스킴 클릭 인터셉트 (http로 접근해야 동작)
  • OWA 자동 — owa.skplanet.com 본문 외부 링크 (여기선 테스트 불가)

4. 클릭 가드 테스트

⚠ click_guard는 http:// 또는 https:// 컨텍스트에서만 동작합니다. file://로 열린 경우 아래 클릭 테스트는 안 됩니다.

4-1. 위험 URI 스킴 — 클릭 시 차단 + confirm 다이얼로그 떠야 함.

applescript://test-inert macOS AppleScript 스킴 (Follina/Pegasus 가족) 차단 + confirm
ms-msdt://test-inert Windows MSDT 스킴 (CVE-2022-30190 Follina) 차단 + confirm
vbscript:msgbox(0) 레거시 VBScript 스킴 차단 + confirm

4-2. 다운로드 트리거 — 클릭 시 click_guard 인터셉트 + scanUrl 호출.

nonexistent_test_file.exe (download attr) EXEC_EXT_RE.exe 매칭 + download 속성 scan 트리거 + 결과 알림
setup_test.dmg EXEC_EXT_RE.dmg 매칭 scan 트리거

4-3. 클립보드 카피 버튼 — 버튼 텍스트가 Copy / 복사 단어이면 click_guard 트리거.

ClickFix 페이로드 (쉘 명령) — click_guard가 scan 시작, 클립보드 후크가 페이로드 캐치 scan + clipboardWrites에 페이로드
정상 카피 (무해한 텍스트) scan 후 SAFE
false-match 회귀 테스트 — 단어 'Copyright'은 trigger 안 되어야 함 트리거 안 됨이 정답
